Unit Testing
What is Unit Testing?
	Unit Testing is the execution of a complete class, routine or small program that has been written by a single programmer or team of programmers, which is tested in isolation from the more complete system.
Unit Testing is conducted by ______________
	Unit Testing is conducted by the developer.
It is type of (blackbox/whitebox testing)
	It is a type of whitebox testing.

Integration Testing
What is Integration Testing?
	Integration Testing is the combined execution of two or more classes, packages, components, or subsystems that have been created by multiple programmers or programming teams.
Integration Testing is conducted by ______________
	Integration Testing is conducted by multiple programmers.
It is type of (blackbox/whitebox testing)
	It may fall under both white box testing and black box testing.

System testing 
What is system Testing?
        System testing is the execution of the software in its final configuration, including integration with other software and hardware systems.
System Testing is conducted by ______________
        System Testing is usually carried out by a team that is independent of the development team in order to measure the quality of the system unbiased.
It is type of (blackbox/whitebox testing)
        White box testing

Acceptance testing
What is acceptance Testing?
        To goal of acceptance testing is to evaluate and validate that the system’s functionality meets the customer’s requirements, standards and business needs before the final delivery
Acceptance Testing is conducted by ______________
        Product Management, Sales and/or Customer Support
It is type of (blackbox/whitebox testing)
        Black box testing
